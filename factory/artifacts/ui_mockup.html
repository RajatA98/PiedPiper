About

An honest audio-embedding pipeline with a published eval.

We embed each track with an open-source audio model (CLAP), then run nearest-neighbour search against a hand-curated reference catalog of roughly 100 recognizable tracks from iTunes previews and 200 Creative Commons songs. Two independent ACRCloud signals — cover-song ID and AI-music detection — run alongside. No black boxes: the detector quality is measured, not claimed, on the evaluation page.

Originally pitched to a confused VC in 2014. Probably more useful now.

Where the noise floor sits

Top-1 cosine similarity for unrelated tracks that should not match anything. The mass clusters well below the cutoff; the threshold is drawn where the distribution’s tail thins out, so a genuinely novel track reads as “completely unique” rather than forcing a spurious match.

Error analysis · named cases with audio

Where it’s wrong, and why

Every failure is a real query/retrieval pair you can listen to. False positives crossed the threshold but aren’t genuine derivatives; false negatives are audible resemblances the embedder scored below the line.

Methodology

How the eval set was built

30 seed songs × 2 Suno generations each gives 60 derivative queries with a known ground-truth source, plus 20–30 unrelated negatives drawn from outside the catalog — roughly 80 tracks total. Recall and MRR are computed on the 60 derivative queries; the score-distribution histogram is computed on the negatives.

Limitations

What this number does not cover

The reference catalog is small (~300 tracks), so coverage is the dominant failure mode — a real source outside the catalog can’t be retrieved. Queries are single-generator (Suno only); Udio and others may shift the score distribution. There is no inter-rater agreement on what counts as “derivative,” and the seed set carries a US-pop bias that likely inflates recall relative to other genres.
